Guide personnel — Projet UF1844

Jugar con varios Tamagotchis · Notes pour comprendre et vérifier l'exercice

1. De quoi parle l'exercice ?

Tu dois créer un jeu de Tamagotchi sur une page web. L'utilisateur choisit un personnage dans une liste, puis prend soin de lui avec des boutons. Ses statistiques baissent automatiquement toutes les 5 secondes. Si la santé et l'énergie tombent toutes les deux à 0, le personnage meurt et la partie s'arrête.

Fichiers obligatoires : 1 HTML + 3 JavaScript (classe, données, logique du jeu). Toutes les fonctions de la logique doivent être des arrow functions.

2. Sélecteur « scalable » (2 points — le plus important)

Ce que ça veut dire : si tu ajoutes un 4ᵉ personnage dans le fichier de données (datos-tamagotchi.js), il doit apparaître tout seul dans le menu déroulant, sans modifier le reste du programme.

Comment on le fait : une boucle for parcourt le tableau tamagotchis et ajoute chaque nom au <select id="seleccionar">.

✓ Dans ton projet : fait dans js/app.js (lignes 36–38). 

for (let i = 0; i < tamagotchis.length; i++) {
  SELECCIONAR.innerHTML += `<option>${tamagotchis[i].nombre}</option>`;
}

Test : ajoute un objet dans personajes → il doit apparaître dans le select sans toucher à app.js.

3. Ce qui doit s'afficher pendant la partie

L'énoncé demande d'afficher :

  1. Le nom
  2. L'image
  3. La description
  4. Les 4 barres de vie : Felicidad, Salud, Limpieza, Energía (valeurs 0–10)

L'énoncé ne impose pas un ordre visuel précis. Il suffit que tout soit visible et se mette à jour à chaque changement.

⚠ Ordre actuel dans juego.html : d'abord les 4 barres (SVG), puis nom → image → description. C'est valide. Si tu préfères l'ordre « nom, image, description, barres », il faudrait inverser les blocs HTML.

✓ Tout est affiché et mis à jour par actualizarPantalla() dans app.js.

4. Valeurs au départ et pendant le jeu

  • À la création : chaque stat reçoit une valeur aléatoire entre 1 et 10.
  • Pendant le jeu : les valeurs restent toujours entre 0 et 10 (ni plus, ni moins).
  • L'écran se met à jour à chaque action et toutes les 5 secondes.

✓ Fait dans clase-tamagotchi.js (constructeur + cambiaValor).

5. Les 6 boutons et leurs effets

Bouton Effet (comme l'énoncé) Ton code
Alimentar +3 énergie, +2 bonheur, −1 propreté ✓
Jugar +2 bonheur, −2 énergie, −2 propreté ✓
Dormir +5 énergie, +2 santé ✓
Duchar +3 santé, propreté = 10 ✓
Reprender −3 bonheur, −2 santé ✓
Acariciar +4 bonheur ✓

Ordre des boutons dans juego.html : Alimentar, Jugar, Duchar, Acariciar, Dormir, Reprender. L'énoncé ne impose pas cet ordre — seulement que les 6 existent.

6. Le timer (toutes les 5 secondes)

Toutes les 5 secondes, la méthode restarVida() enlève 1 point à chaque stat (bonheur, santé, propreté, énergie), puis l'écran se met à jour.

✓ Fait avec setInterval(..., 5000) dans iniciarTimer().

7. La mort (Game Over)

Condition de mort : salud === 0 ET energia === 0 en même temps.

Quand ça arrive, il faut :

  • Arrêter le timer → clearInterval(temporizador)
  • Enlever les écouteurs des boutons → quitarEscuchas()
  • Le jeu est fini (boutons désactivés + message Game Over)

✓ Tout cela est dans finPartida() et comprobarVida().

index.html et guia.html sont des extras pour toi, pas demandés par l'énoncé.

9. La classe Tamagotchi (détail)

Publics : nombre, descripcion, imagen

Privés : felicidad, salud, limpieza, energia, enVida (commence à true)

Chaque interaction appelle comprobarVida() à la fin pour vérifier si le personnage est encore vivant.

✓ Ta classe dans clase-tamagotchi.js respecte l'énoncé.

Extra (non demandé) : videoSaludo et videos pour les animations.

11. Checklist pas à pas (à cocher avant de rendre)

  1. Ouvrir juego.html dans le navigateur (pas seulement index.html).
  2. Sélection : le <select id="seleccionar"> montre les 3 personnages.
  3. Scalable : ajouter un 4ᵉ dans datos-tamagotchi.js → il apparaît dans le select sans toucher app.js.
  4. Continuar : cliquer démarre la partie, cache la sélection, montre le jeu.
  5. Affichage : nom, image, description et 4 stats visibles.
  6. Stats initiales : valeurs entre 1 et 10 (pas toutes à 0).
  7. Chaque bouton : tester les 6 et vérifier que les stats changent comme le tableau ci-dessus.
  8. Timer : attendre 5 s sans cliquer → toutes les stats perdent 1 point.
  9. Mort : faire descendre salud ET energia à 0 → Game Over, timer arrêté, boutons inactifs.
  10. Listeners : après la mort, cliquer sur un bouton d'action ne doit rien faire.
  11. Fichiers : 1 HTML (juego.html) + 3 JS avec scripts en defer dans le bon ordre.
  12. Arrow functions : toutes les fonctions dans app.js utilisent const nom = () =>.
  13. Responsive : réduire la fenêtre → la mise en page reste lisible (mobile).
  14. Console : F12 → pas d'erreurs rouges au chargement ni en jouant.
